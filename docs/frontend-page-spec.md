# Pipeline Studio 前端页面功能说明（供设计稿）

本文档描述当前原型 `frontend/index.html` 的信息架构、页面区块、交互与数据关系，便于专业人士绘制高保真界面。实现技术为 Vue 3（CDN），样式为自定义深色控制台风格。

---

## 1. 产品定位

- **页面名称**：AgentTeam Matrix / Pipeline Studio（流水线编辑台）。
- **目标用户**：研发流程编排人员（配置流水线、阶段、Agent 职责与 Skill 装配）。
- **核心任务**：在单页应用内完成「流水线 → 阶段 → Agent → Skill」的层级配置，左侧为功能导航，右侧为当前功能的工作区。

---

## 2. 整体布局（信息架构）

| 区域 | 说明 |
|------|------|
| **左侧边栏（固定宽度）** | 品牌区 + 主导航菜单（4 项），点击切换右侧内容，当前项高亮。 |
| **右侧主工作区** | 顶部为当前菜单标题 + 上下文条（当前选中的流水线名称）；下方为与当前菜单对应的单块面板（`panel`）。 |

**主导航菜单项（自上而下，顺序固定）**

1. **流水线管理**（key: `pipeline`）
2. **阶段编排**（key: `stage`）
3. **Agent 职责**（key: `agent`）
4. **Skill 管理**（key: `skill`）

**特殊交互**：在「流水线管理」中点击某条流水线后，会自动将左侧激活菜单切换为「阶段编排」，引导用户继续配置阶段。

---

## 3. 数据模型（便于理解页面元素）

以下为逻辑结构，设计时可表现为列表、卡片、树等。

```
Pipeline（流水线）
├── id, name
└── Stage[]（阶段，有序列表）
    ├── id, name
    └── Agent[]（该阶段下的 Agent）
        ├── id, name
        ├── responsibility（职责长文本，可编辑）
        └── Skill[]（挂载在该 Agent 上的 Skill）
            ├── id, name
            └── version
```

- 同一时间有 **0～N 条流水线**，其中 **最多一条为「当前选中」**（`selectedPipeline`）。
- 所有「阶段 / Agent / Skill」操作均针对 **当前选中的流水线**。

---

## 4. 右侧顶部栏（工作区 Header）

| 元素 | 内容 | 说明 |
|------|------|------|
| 主标题 | 与左侧当前菜单一致 | 如「流水线管理」「阶段编排」等 |
| 副标题/上下文 | `当前流水线：{名称}` 或 `未选择` | 全局提示当前操作上下文 |

---

## 5. 各菜单页功能明细

### 5.1 流水线管理（`pipeline`）

**目的**：创建流水线、在多条流水线中选中一条作为编辑目标。

| UI 元素 | 行为 |
|---------|------|
| 文本输入框 | 占位提示示例：「新流水线名称，例如：核心研发流程」 |
| 「创建」按钮 | 名称非空则新增一条流水线，并自动设为当前选中；清空输入框 |
| 流水线列表 | 每项展示：标题（流水线名称）、副文案（Stages: {数量}）；点击整行选中该流水线，并 **自动跳转左侧菜单至「阶段编排」** |

**空状态**：无流水线时列表为空，仅显示输入区与创建按钮。

---

### 5.2 阶段编排（`stage`）

**目的**：为当前选中的流水线添加阶段，并查看阶段列表及每阶段 Agent 数量。

**前置条件**：必须先选中一条流水线；否则展示提示文案：「请先在「流水线管理」里创建或选择一条流水线。」

| UI 元素 | 行为 |
|---------|------|
| 文本输入框 | 占位：「阶段名称，例如：技术方案」 |
| 「添加阶段」按钮 | 在当前流水线末尾追加一个阶段；成功后清空输入框；若 `agentStageId` / `skillStageId` 未设置，会将新阶段设为默认关联（供后续页使用） |
| 阶段列表 | 每项：阶段名称、副文案（Agents: {数量}） |

**说明（产品层）**：当前原型 **不支持** 阶段拖拽排序、重命名、删除；若设计稿需要，可作为后续迭代标注。

---

### 5.3 Agent 职责（`agent`）

**目的**：按阶段添加 Agent，并为每个 Agent 编辑职责长文。

**前置条件**：当前流水线存在且至少有一个阶段；否则提示：「请先添加阶段，再开始编辑 Agent 职责。」

| UI 元素 | 行为 |
|---------|------|
| 下拉框 | 选择目标阶段（来自当前流水线的阶段列表） |
| 文本输入 | Agent 名称 |
| 文本输入 | 职责描述（占位提示：目标、边界、交付标准） |
| 「添加 Agent」按钮 | 在选定阶段下新增 Agent；成功后清空名称与职责输入；若 Skill 页当前阶段与添加阶段一致，会自动选中新 Agent |

**下方展示区**：按 **阶段** 分卡片（`board-card`），每卡内：

- 阶段标题
- 若该阶段无 Agent：展示「暂无 Agent」
- 若有：每个 Agent 一块，含 **标签（Agent 名称）+ 多行文本框（职责，双向绑定可改）**

---

### 5.4 Skill 管理（`skill`）

**目的**：在「阶段 → Agent」维度上为 Agent 装配 Skill，并查看该 Agent 已装配列表。

**前置条件**：当前流水线有阶段；否则提示先创建阶段与 Agent。

| UI 元素 | 行为 |
|---------|------|
| 下拉框 1 | 选择阶段（决定下方 Agent 列表来源） |
| 下拉框 2 | 选择该阶段下的 Agent |
| 文本输入 | Skill 名称（示例：risk_extractor） |
| 文本输入 | 版本（默认占位 1.0.0） |
| 「装配 Skill」按钮 | 将 Skill 追加到所选 Agent；成功后清空 Skill 名称输入 |

**已装配列表**：当选中某个 Agent 时，以列表展示其 `skills`，每项：Skill 名称 + 版本号。

**空状态**：

- 未选 Agent：「请选择 Agent 后查看或装配 Skill。」
- 无阶段/Agent：「请先创建阶段和 Agent，再进行 Skill 装配。」

## 6. 校验与禁用规则（原型现状）

| 操作 | 条件不满足时 |
|------|----------------|
| 创建流水线 | 名称为空则不创建（无显式错误提示，静默） |
| 添加阶段 | 未选流水线或名称为空则不添加 |
| 添加 Agent | 未选流水线、未选阶段、名称为空则不添加 |
| 装配 Skill | 未选流水线、阶段、Agent，或 Skill 名称为空则不装配 |

设计时可考虑补充：toast、内联错误、禁用态按钮等（当前原型未做）。

---

## 7. 设计建议（非强制）

- **左侧导航**：可考虑图标 + 文案；当前选中态需与未选中区分明显。
- **流程导向**：用户从「流水线 → 阶段 → Agent → Skill」路径清晰；选中流水线后自动跳到「阶段编排」可改为「仅高亮提示」或保留，需与产品确认。
- **Skill 管理**：后续「平台级 Skill 库」与「装配到 Agent」可能拆成两页，本文档仅描述当前「装配」页。
- **响应式**：当前为桌面优先布局（侧栏 + 主区）；移动端可折叠侧栏为抽屉。

### 8.1 `frontend/test-runner.html`（可视化演示版）

该文件已由独立演进为 **高保真交互演示**：Vue 3 Composition API + Tailwind CSS（CDN）+ Font Awesome，**不再引用** `ui-state.js`。

| 项目 | 说明 |
|------|------|
| 布局 | 左侧边栏菜单 + 中间窄栏表单 + 右侧大面积 **流程画布**（阶段节点、Agent 卡片、Skill 标签、SVG 连线示意） |
| 数据 | 内置示例流水线「核心研发流程」作为初始数据；操作逻辑与主原型类似（创建流水线、阶段、Agent、装配 Skill） |
| 用途 | 供设计/产品快速感受「流水线即流程图」的视觉与信息层级；与 `index.html` 为 **并行两套**，后续可择一合并或抽取公共组件 |

若需恢复 **仅针对 `ui-state.js` 的自动化断言测试**，建议另建独立页面（例如 `frontend/ui-state-tests.html`），避免与演示页混用。
